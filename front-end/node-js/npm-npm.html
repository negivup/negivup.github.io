npm 简介及常用字段 #

npm 不是 Node Package Manager 的简写，而是全称 npm is not an acronym 的缩写。类似的还有 GNU（GNU is Not Unix）、PNG（PNG is Not Gif）等。

“npm is not an acronym” 的汉语翻译是“npm 不是一个缩写”

npm 不用单独安装，在安装 Node.js 后会自动安装。

初始化项目 #

使用 Node.js 管理的项目都会在项目根目录下创建一个 package.json 文件，这个文件是管理项目依赖的核心文件。

package.json 文件在初始化项目的时候生成 :

bash
# 初始化项目，需要回答问题
$ npm init

# 初始化项目，跳过问题
$ pnpm init -y

常用字段 #

package.json 中的字段说明可以查看 package.json 指南。

package-lock.json 的作用是锁定版本，具体说明可以参考 package-lock.json 文件。

name

name 项目的名称，如 "name": "xxxx"。

version

version 项目的版本号，如 "version": "x.y.z"。

description

description 项目的描述，作为项目的基本描述，如 "description": "xxxx"。

author

author 项目的作者信息，一般在发布时使用，如 "author": "name <email@xxx.com>"。

license

license 项目的开源协议，一般在发布时使用，如 "license": "MIT"。

private

private 指定当前项目是否是私有项目，如果是公司项目，不想因为失误发布到公网上，可以设置 "private": true。

engines

engines 用于指定 node、npm、os 等依赖的版本，在安装依赖时会先检查对应的引擎版本，如果不符合会报错，如 "engines": { "node": ">=14.0.0", "pnpm": ">=7.13.0" }。

keywords

keywords 用于指定当前项目的关键字，一般在发布时使用，方便搜索，如 "keywords": ["business", "dealer", "management"]。

files

files 用于指定发布当前项目时需要上传的文件及文件夹，如 "files": ["dist", "main.js"]。

homepage

homepage 用于设置项目的官网地址，如 "homepage": "https://www.baidu.com"。

repository

repository 用于设置项目源码所在的仓库，如 "repository": { "type": "git", "url": "xxxxxxxx.git" }。

scripts

scripts 用于配置一些脚本命令，以键值对的形式存在，通过 npm run xxx 执行。

dependencies

dependencies 用于保存生产依赖，这里面的依赖不管是在开发环境还是在生产环境都会使用，一般通过 npm install --save 自动维护。

devDependencies

devDependencies 用于保存开发依赖，这里面的依赖不管只在开发环境中会使用，一般通过 npm install --save-dev 自动维护。

peerDependencies

peerDependencies 如果安装的依赖需要先安装另一个依赖才能正常运行，如 Vue 组件库一般会通过 peerDependencies 指定其需要 vue 才能运行，如 "peerDependencies": { "vue": "^2.6.14" }。

main

main 用于设置程序的入口，如 "main": "index.js"，在程序中加载需要使用 require 加载。
