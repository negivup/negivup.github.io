CommonJS 模块 #

CommonJS 模块 是 Node.js 中的模块加载方式。Node.js 具有内置的模块系统。

在模块中，默认情况下，定义的任何对象或变量都是私有的，不会在外部暴露。每个文件就一个模块，有自己的作用域。在一个文件里面定义的变量、函数、类，都是私有的，对其他文件不可见。

  1. 私有作用域不会污染全局作用域
  2. 模块可加载多次，只会在第一次加载时运行一次
  3. 模块加载顺序是按照其在代码中出现的顺序

基础使用 #

module.exports #

一般使用 module.exports 将模块导出，可以导出单个变量，也可以导出一个对象。

1.导出单个变量

js
// foo.js
function add(num1, num2) {
  return num1 + num2;
}
module.exports = add;

2.导出一个变量

js
// bar.js
function sum(num1, num2) {
  return num1 + num2;
}
function getNum(num) {
  return num;
}
module.exports = { sum, getNum };

3.引入模块

js
// index.js
const add = require('./foo.js');
const { sum, getNum } = require('./bar.js');
console.log(add(1, 2));
console.log(sum(2, 2));
console.log(getNum(2));

exports #

导出模块除了使用 module.exports 的方式之外，还可以使用 exports 的方式。

1.导出单个变量

js
// foo.js
function add(num1, num2) {
  return num1 + num2;
}
exports.add = add;

2.导出多个变量

js
// bar.js
function sum(num1, num2) {
  return num1 + num2;
}
function getNum(num) {
  return num;
}
exports.sum = sum;
exports.getNum = getNum;

3.引入模块

js
// index.js
const foo = require('./foo.js');
const bar = require('./bar.js');
console.log(foo);
console.log(bar);

4.执行程序

bash
$ node index.js
{ add: [Function: add] }
{ sum: [Function: sum], getNum: [Function: getNum] }

深入理解 #

如果使用 exports 导出多个变量，使用下面的方式：

js
// bar.js
function sum(num1, num2) {
  return num1 + num2;
}
function getNum(num) {
  return num;
}
exports = { sum, getNum };

在执行程序的时候会发现，导出的内容是一个空对象。

js
// index.js
const bar = require('./bar.js');
console.log(bar); // {}

这是因为 exports 其实是 module.exports 的快捷方式 exports= module.exports = {}，只有它们指向同一个内存地址的时候，才是等价的。

在 Node.js 中每个 js 文件中都有两个内置的对象：module 和 exports。

js
// index.js
console.log(module.exports);
console.log(exports);

当更改其中一个指向的时候，会发现两个值就不同了，它们指向的内存就发生变化了。

js
// index.js
exports = { b: 2 };
console.log(module.exports); // {}
console.log(exports); // { b: 2 }
js
// index.js
module.exports = { a: 1 };
console.log(module.exports); // { a: 1 }
console.log(exports); // {}

如果想让它们重新建立关系，可以使用 exports = module.exports。

js
// index.js
module.exports = { a: 1 };
exports = module.exports;
console.log(module.exports); // { a: 1 }
console.log(exports); // { a: 1 }

总结：

  • exports 是 module.exports 的一个引用，内部实现原理 exports = module.exports = {};
  • require 引用模块后，返回给调用者的是 module.exports 而不是 exports
  • exports.xxx 相当于在导出对象上挂属性，该属性对调用模块直接可见
  • exports = xxx 相当于给 exports 对象重新赋值，调用模块不能访问 exports 对象及其属性
