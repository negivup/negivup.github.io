简介 #

Node.js 的核心是事件驱动编程，程序无法控制用户是否点击或何时点击一个页面元素，程序只能在用户点击了某个元素再处理对应的事件。

在用户界面中很好理解，用户操作某个页面元素，程序做对应的操作；在服务器上，用户触发的事件就是一个 HTTP 请求。

一旦开始从事件驱动编程的角度思考问题，会发现到处都是事件。从一个页面导航到另一个页面，或从一块内容导航到另一块内容，都是事件。

Node.js 跟传统 Web 服务器一个显著的差异是 : Node.js 是单线程的。单线程极大地简化了 Web 应用的编程，而当需要达到多线程应用的性能时，只需简单地多开几个 Node 实例，就能获得多线程的好处。通过服务器并行（跟应用内的并行相对）实现的多线程，只是把复杂性转移。

Node.js 是平台独立的，在各大主流操作系统（Windows、macOS 和 Linux）上搭建 Node 都轻而易举，不同操作系统协作起来也很容易。

Hello World #

比起传统的 Web 服务器，Node 提供了一种不同的范式 : 你所写的应用，其本身就是 Web 服务器。Node 只是给你提供了构建 Web 服务器的框架。

创建一个 helloworld.js 文件，并编写如下代码，使用的是 Node.js 的内置 http 模块（详细使用可以查看 Node.js 文档） :

js
const http = require('http');
const port = process.env.PORT || 3000;

// 创建一个 HTTP 服务器
const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('Hello world!');
});

// 启动服务并监听 3000 端口
server.listen(port, () => {
  console.log(`服务访问地址 : http://localhost:${port}`);
});

在终端中进入到 helloworld.js 所在的目录，然后输入 :

bash
$ node helloworld.js

打开浏览器，地址栏中输入 http://localhost:3000，就可以在浏览器中看到文本 Hello world!。

路由 #

应用如何响应导航事件，就叫作路由，路由指的是对客户端内容请求的满足机制。

下面在 helloworld.js 中完善代码，对首页、关于和 404 页面路由做处理 :

js
const http = require('http');
const port = process.env.PORT || 3000;

const server = http.createServer((req, res) => {
  // 对 URL 做规范化处理：移除查询串、末尾可选的斜杠，并转成小写
  const path = req.url.replace(/\/?(?:\?.*)?$/, '').toLowerCase();
  switch (path) {
    case '':
      res.writeHead(200, { 'Content-Type': 'text/plain' });
      res.end('Homepage');
      break;
    case '/about':
      res.writeHead(200, { 'Content-Type': 'text/plain' });
      res.end('About');
      break;
    default:
      res.writeHead(404, { 'Content-Type': 'text/plain' });
      res.end('Not Found');
      break;
  }
});

server.listen(port, () => {
  console.log(`服务访问地址 : http://localhost:${port}`);
});

通过 node helloworld.js 运行程序，浏览 http://localhost:3000 会展示 Homepage，浏览 http://localhost:3000/about 会展示 About，浏览其他地址（如 http://localhost:3000/foo）都会展示 Not Found。

由于使用了正则 /\/?(?:\?.*)?$/ 对 URL 做处理，因此任何查询字符串都会被忽略，http://localhost:3000/?foo=bar 会展示 Homepage。

静态资源 #

静态资源一般是不会变化的内容，如图片、媒体文件等。

一般情况下，会把静态资源放在 public 文件夹中。存放 home.html、about.html、404.html 和 img/logo.png。

home.html

html
<h1>Homepage</h1>
<p><img src="/img/logo.png" alt="logo" /></p>

about.html

html
<h1>About</h1>
<p><img src="/img/logo.png" alt="logo" /></p>

404.html

html
<h1>Not Found</h1>
<p><img src="/img/logo.png" alt="logo" /></p>

1.创建一个辅助函数处理静态资源

js
const fs = require('fs');

function serveStaticFile(res, path, contentType, responseCode = 200) {
  // fs.readFile 是一个读取文件的异步方法
  // fs.readFile 对应的同步方法 fs.readFileSync
  fs.readFile(__dirname + path, (err, data) => {
    if (err) {
      res.writeHead(500, { 'Content-Type': 'text/plain' });
      return res.end('500 - Internal Error');
    }
    res.writeHead(responseCode, { 'Content-Type': contentType });
    res.end(data);
  });
}

2.创建 HTTP 服务器时读取静态资源

js
const server = http.createServer((req, res) => {
  // 对 URL 做规范化处理：移除查询串、末尾可选的斜杠，并转成小写
  const path = req.url.replace(/\/?(?:\?.*)?$/, '').toLowerCase();
  switch (path) {
    case '':
      serveStaticFile(res, '/public/home.html', 'text/html');
      break;
    case '/about':
      serveStaticFile(res, '/public/about.html', 'text/html');
      break;
    case '/img/logo.png':
      serveStaticFile(res, '/public/img/logo.png', 'image/png');
      break;
    default:
      serveStaticFile(res, '/public/404.html', 'text/html', 404);
      break;
  }
});

通过 node helloworld.js 运行程序，输入对应的地址就可以在浏览器中看到对应的页面。
