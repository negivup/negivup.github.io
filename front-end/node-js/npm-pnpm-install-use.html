pnpm 的安装及使用 #

安装 #

安装 pnpm 可以使用一种简单的安装方式，通过 npm 全局安装 :

bash
$ npm install -g pnpm

安装完成之后，需要将 pnpm 注册成全局脚本 :

bash
$ pnpm setup

之后可以检测一下是否安装成功 :

bash
$ pnpm -v
7.13.5

使用 #

为了方便与 npm 使用方式做对比，下面列举一下常用的使用方式 :

npm 命令pnpm 命令说明
npm installpnpm install初始化项目的依赖
npm install --save <pkg>pnpm install --save <pkg>安装生产依赖
npm install --save-dev <pkg>pnpm install --save-dev <pkg>安装开发依赖
npm install -g <pkg>pnpm install -g <pkg>安装全局依赖
npm install --save-exact <pkg>pnpm install --save-exact <pkg>安装确切版本的依赖
npm uninstall <pkg>pnpm remove <pkg>卸载依赖
npm run <cmd>pnpm run <cmd>运行 scripts 中的指令
npx <cmd>pnpm exec <pkg>运行 node_modules 中的指令

缓存 #

pnpm 中文件的统一存储位置以 7.0 版本为分界线，发生了一些变化 :

  • 7.0 之前 : ~/.pnpm-store
  • 7.0 之后 :
    • Linux : ~/.local/share/pnpm/store
    • Windows : %LOCALAPPDATA%/pnpm/store
    • Mac : ~/Library/pnpm/store

~ 一般指的是用户目录，打开方式如下 :

  • Windows 中快速打开 : Win + R，输入 . 或 %USERPROFILE%，点击确定
  • Mac 中快速打开 : 打开终端，输入 open ~

如果想获取当前活跃的 store 目录，可以使用下面的指令 :

bash
# 如果项目的配置文件 .npmrc 指定了 store-dir 字段，那么获取的是这个值
# 如果没有指定，获取的是全局的 store 目录
$ pnpm store path

如果确定一些包在当前电脑中没有项目使用了，或者想清理一些 store 目录下当前未被引用的依赖，可以使用下面的命令释放 store 所占的存储空间 :

bash
$ pnpm store prune

卸载 #

卸载 pnpm 比较简单，直接使用全局卸载的方式即可 :

bash
$ npm uninstall -g pnpm

在卸载完成之后，重点是要清空 pnpm 在电脑中缓存信息。

  1. 删除 ~/.pnpm-store 文件夹
  2. 删除 %LOCALAPPDATA%/pnpm、~/.local/share/pnpm、~/Library/pnpm 文件夹
  3. 删除 %LOCALAPPDATA%/pnpm-cache、~/.local/share/pnpm-cache、~/Library/pnpm-cache 文件夹
  4. 删除 %LOCALAPPDATA%/pnpm-state、~/.local/share/pnpm-state、~/Library/pnpm-state 文件夹
  5. 删除通过 .npmrc 中 store-dir 字段指定的文件夹
