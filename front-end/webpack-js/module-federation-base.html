简单使用 #

模块联邦（Module Federation）可以将多个单独构建的应用组合成一个应用。独立的应用之间没有相互依赖关系，可以独立开发独立部署。

这通常被称为微前端（Micro-Frontends），但模块联邦（Module Federation）不局限于此。下面先来看一个基础的例子，体验一下模块联邦（Module Federation）。

下面我们用一个简单的例子来看一下如何使用模块联邦（Module Federation）。

直接下载源代码，安装依赖，运行程序就可以看到效果：

bash
$ pnpm install
# 打开两个控制台，同时运行下面两个命令
$ pnpm run app1:dev
$ pnpm run app2:dev

基础环境 #

为了方便管理依赖，这个例子使用 pnpm 作为包管理工具。并用 workspace 管理依赖。

创建文件夹并初始化项目
bash
$ mkdir module-federation && cde module-federation && pnpm init -y

为了能够快速拉取依赖，配置一下镜像源：

.npmrc
registry=https://registry.npmmirror.com/
shamefully-hoist=true

模块联邦（Module Federation）需要多个可以单独打包和独立部署的应用，为了方便管理依赖这里使用 workspace。

创建 Monorepo 的管理文件 pnpm-workspace.yaml：

pnpm-workspace.yaml
yaml
packages:
  - 'packages/**'
安装开发依赖
bash
$ pnpm add -w --save-dev --save-exact @babel/core @babel/preset-react babel-loader external-remotes-plugin html-webpack-plugin webpack webpack-cli webpack-dev-server

依赖的版本信息如下：

devDependencies:
+ @babel/core 7.18.2
+ @babel/preset-react 7.17.12
+ babel-loader 8.2.5
+ external-remotes-plugin 1.0.0
+ html-webpack-plugin 5.5.0
+ webpack 5.73.0
+ webpack-cli 4.9.2
+ webpack-dev-server 4.9.1

为了方便后期开发，增加几个 npm 脚本：

npm 脚本
bash
$ npm set-script app1:dev "cd packages/app1 && pnpm run dev"
$ npm set-script app2:dev "cd packages/app2 && pnpm run dev"
$ npm set-script app1:build "cd packages/app1 && pnpm run build"
$ npm set-script app2:build "cd packages/app2 && pnpm run build"

创建 app2 需要的文件夹并创建 package.json：

package.json
bash
$ mkdir packages && mkdir packages/app2 && cd packages/app2
$ echo "{\"name\":\"app2\"}" > package.json
$ npm set-script dev "webpack serve --mode=development"
$ npm set-script build "webpack --mode=production"
$ cd ../../

创建 app1 需要的文件夹并创建 package.json：

package.json
bash
$ mkdir packages/app1 && cd packages/app1
$ echo "{\"name\":\"app1\"}" > package.json
$ npm set-script dev "webpack serve --mode=development"
$ npm set-script build "webpack --mode=production"
$ cd ../../
安装依赖
bash
$ pnpm add --filter app1 --filter app2 --save --save-exact react@17.0.2 react-dom@17.0.2

app2 #

在 packages/app2 中创建一个 Webpack 的配置文件 webpack.config.js：

webpack.config.js
js
const HtmlWebpackPlugin = require('html-webpack-plugin');
const { ModuleFederationPlugin } = require('webpack').container;

const templateContent = ({ htmlWebpackPlugin }) => `
<!DOCTYPE html>
<html lang="en">
  <head>
    <title>${htmlWebpackPlugin.options.title}</title>
    <meta charset="UTF-8" />
  </head>
  <body>
    <div id="root"></div>
  </body>
</html>
`;

/** @type {import('webpack').Configuration} */
const webpackConfig = {
  entry: './src/index.js',
  devServer: { port: 3002 },
  module: {
    rules: [
      {
        test: /\.jsx?$/,
        loader: 'babel-loader',
        exclude: /node_modules/,
        options: {
          presets: ['@babel/preset-react'],
        },
      },
    ],
  },
  plugins: [
    new ModuleFederationPlugin({
      name: 'app2',
      filename: 'remoteEntry.js',
      exposes: {
        './App': './src/App',
      },
      shared: {
        'react': { singleton: true },
        'react-dom': { singleton: true },
      },
    }),
    new HtmlWebpackPlugin({ title: 'App 2', templateContent }),
  ],
};

module.exports = webpackConfig;
src/index.js
js
import('./bootstrap.js');
src/bootstrap.js
js
import React from 'react';
import ReactDOM from 'react-dom';

import App from './App.js';

ReactDOM.render(<App />, document.getElementById('root'));
src/App.js
js
import React from 'react';

const App = () => {
  return (
    <div
      style={{
        margin: '10px',
        padding: '10px',
        textAlign: 'center',
        backgroundColor: 'cyan',
      }}
    >
      <h1>App 2</h1>
    </div>
  );
};

export default App;
执行命令启动程序
bash
$ pnpm run app2:dev

app1 #

在 packages/app1 中创建一个 Webpack 的配置文件 webpack.config.js：

webpack.config.js
js
const HtmlWebpackPlugin = require('html-webpack-plugin');
const { ModuleFederationPlugin } = require('webpack').container;
const ExternalTemplateRemotesPlugin = require('external-remotes-plugin');

const templateContent = ({ htmlWebpackPlugin }) => `
<!DOCTYPE html>
<html lang="en">
  <head>
    <title>${htmlWebpackPlugin.options.title}</title>
    <meta charset="UTF-8" />
  </head>
  <body>
    <div id="root"></div>
  </body>
</html>
`;

/** @type {import('webpack').Configuration} */
const webpackConfig = {
  entry: './src/index.js',
  devServer: { port: 3001 },
  module: {
    rules: [
      {
        test: /\.jsx?$/,
        loader: 'babel-loader',
        exclude: /node_modules/,
        options: {
          presets: ['@babel/preset-react'],
        },
      },
    ],
  },
  plugins: [
    new ModuleFederationPlugin({
      name: 'app1',
      remotes: {
        app2: 'app2@[window.app2Url]/remoteEntry.js?[getRandomString()]',
      },
      shared: {
        'react': { singleton: true },
        'react-dom': { singleton: true },
      },
    }),
    new ExternalTemplateRemotesPlugin(),
    new HtmlWebpackPlugin({ title: 'App 1', templateContent }),
  ],
};

module.exports = webpackConfig;
src/index.js
js
// 可以在此处编写具体的逻辑来获取最终的 URL
window.app2Url = 'http://localhost:3002';
window.getRandomString = () => `r=${new Date().getTime()}`;

// 在这里使用动态导入的方式，允许 Webpack 和模块联邦的代码进行交互
import('./bootstrap.js');
src/bootstrap.js
js
import React from 'react';
import ReactDOM from 'react-dom';

import App from './App.js';

ReactDOM.render(<App />, document.getElementById('root'));
src/App.js
js
import React, { Suspense } from 'react';
const RemoteApp = React.lazy(() => import('app2/App'));

const App = () => {
  return (
    <div>
      <div
        style={{
          margin: '10px',
          padding: '10px',
          textAlign: 'center',
          backgroundColor: 'greenyellow',
        }}
      >
        <h1>App1</h1>
      </div>
      <Suspense fallback={'loading...'}>
        <RemoteApp />
      </Suspense>
    </div>
  );
};

export default App;
同时运行下面的命令
bash
$ pnpm run app2:dev
$ pnpm run app1:dev

注意

生产部署时，app2 的部署地址可以调整，对应的 app1 中的 app2Url 的地址也要调整成 app2 部署的地址，否则程序将无法正常运行。
